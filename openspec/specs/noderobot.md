# noderobot 项目规格文档

## 1. 项目概述

**项目名称**: noderobot  
**项目类型**: 订阅合并与节点管理工具  
**核心功能**: 从多个远程订阅源获取代理节点，进行清洗、去重、GeoIP 重命名后输出 Clash 和 V2RayN 格式的配置文件  
**目标用户**: 需要整合多个代理订阅源的用户

## 4. 功能模块说明

### 4.1 订阅源管理 (source_manager.py)
- 从 `sources.json` 读取订阅源配置
- 下载远程订阅内容
- 计算 SHA256 校验和，判断是否有更新
- 解析 YAML，提取节点列表

### 4.2 节点解析 (parser.py)
- 解析多种格式的节点链接 (ss://, vmess://, trojan://, vless://)
- 支持从 YAML 文件中提取节点

### 4.3 过滤与清洗 (filters.py, cleaner.py)

#### cleaner.py (iPhone 订阅中转专用)
**用途**: iPhone 订阅中转，解决 clashfree 动态文件名问题

**问题背景**:
- clashfree 的订阅文件名每天都会变动 (如 `freenodes-2024-01-01.yml`)
- iPhone 的订阅软件只能配置固定 URL，无法读取动态命名的订阅
- 通过 GitHub Actions 中转，输出固定文件名的 `merge.yml`

**清洗逻辑**:
1. **SS 无效节点过滤**: 移除 `type: ss` 且 `cipher: ss` 的无效节点
2. **代理组白名单过滤**: 只保留特定的代理组:
   - `🔰 节点选择`
   - `♻️ 自动选择`
   - `🎯 全球直连`
   - `🐟 漏网之鱼`
3. **代理引用清理**: 移除对已删除节点的引用

#### filters.py (通用过滤)
- **IP 黑名单过滤**: 过滤 `ip-block-list.txt` 中定义的 IP
- **HTTP 协议过滤**: 移除 type=http 的节点

### 4.4 去重与排序 (proxy_tools.py)
- 基于 (server, port) 去重
- 支持手动节点 (名称含 `|M|`) 优先级最高
- 按国家代码和使用次数排序

### 4.5 GeoIP 重命名 (geoip.py)
- 查询 IP 归属地 (国家/城市)
- 生成标准化名称: `🇺🇸 US|美国 01`
- 使用 `Country.mmdb` 和 `City.mmdb` 数据库

### 4.6 节点统计 (csvtool.py)
- 记录每个服务器 IP 的使用次数
- 持久化到 `node-server-statistics.csv`
- 用于生成推荐排序

### 5.2 插件功能说明

| 插件 | 功能 | 关键逻辑 |
|------|------|----------|
| **0-julong.js** | 工具库 | 网络检测 (IPv4/IPv6/URL)、CSV 操作、性能监控、日志 |
| **1.1-filter.js** | 条件过滤 | 支持 regex/contains/equals/ipv6 四种匹配模式，AND/OR 逻辑 |
| **2-quchong.js** | 去重 | 基于 server:port:type 去重，保留最新节点 |
| **3-rename-ip-count.js** | 使用统计 | 记录节点使用次数，名称添加 `N C|` 前缀 |
| **4-speedtest.js** | 测速 | ICMP ping，并发控制 (默认10)，记录通过率到 CSV |
| **4.2-rename-add-pass-count.js** | 测速统计 | 结合测速结果，添加 `通过率%|N C|` 格式前缀 |
| **5-filter-ping-rate.js** | 通过率过滤 | 过滤通过率 < N% 且次数 >= M 的节点 |

### 5.3 节点命名约定 (sub-store)

```
{通过率}%|{使用次数}C|{原名称}

示例:
95%|10C|🇺🇸 US|美国 01
│   │  └────────────── 使用次数 (10次)
│   └─────────────── 通过率 (95%)
└────────────────── 格式前缀
```

## 7. 自动化 (GitHub Actions)

### merge.yml 工作流
- **触发**: 每小时执行 (`5 * * * *`) 或手动触发
- **功能**:
  1. 下载最新 GeoIP 数据库
  2. 执行 cleaner.py 清洗
  3. 执行 merge.py 合并
  4. 检测文件变化
  5. 自动提交推送
  6. 发送 Telegram 通知

### conn.yml 工作流
- 用于连接性测试

## 8. 节点命名规范

```
{国旗} {国家代码}|{国家名}[-{城市名}] {序号:02d}

示例:
🇺🇸 US|美国 01
🇸🇬 SG|新加坡-新加坡 02
🇭🇰 HK|香港-九龙 03
🌐 XX 01 (未知地区)
```

### 特殊节点
- 手动节点: 名称包含 `|M|`
- 时间戳节点: 名称包含 `-Timestamp`

## 10. 注意事项

1. **GeoSite.dat** 由 Clash 内核运行时加载，无需在 Python 侧处理
2. **sub-store** 子项目已与主项目分离，无数据交互
3. **SHA256 校验** 用于判断订阅是否有更新，避免无意义重写
4. **手动节点** 优先级最高，会排除同名 IP 的自动节点

## 11. 关键设计决策

### 11.1 clashfree 动态文件名问题
- **问题**: clashfree 订阅文件名每天变动，iPhone 客户端只能配置固定 URL
- **解决方案**: 使用 GitHub Actions 定时运行 cleaner.py，输出固定文件名 `merge.yml`
- **触发频率**: 每小时执行 (`5 * * * *`)

### 11.2 双系统并存
- **原因**: iPhone 无法运行 sub-store，需要 Python + GitHub Actions 中转
- **现状**: Mac 已迁移到 sub-store，iPhone 仍使用 Python 中转
- **无整合计划**: 两套系统独立运行，无数据交互
