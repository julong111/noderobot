# noderobot 项目规格文档

## 1. 项目概述

**项目名称**: noderobot  
**项目类型**: 订阅合并与节点管理工具  
**核心功能**: 从多个远程订阅源获取代理节点，进行清洗、去重、GeoIP 重命名后输出 Clash 和 V2RayN 格式的配置文件  
**目标用户**: 需要整合多个代理订阅源的用户

### 技术栈
- **语言**: Python 3.x
- **依赖**: requests, pyyaml, geoip2
- **外部工具**: mihomo (Clash.Meta 内核)
- **自动化**: GitHub Actions
- **子项目**: sub-store (Node.js, 已分离)

### 支持平台
- Windows
- macOS
- iOS (通过 Clash 客户端)

## 4. 功能模块说明

### 4.1 订阅源管理 (source_manager.py)
- 从 `sources.json` 读取订阅源配置
- 下载远程订阅内容
- 计算 SHA256 校验和，判断是否有更新
- 解析 YAML，提取节点列表

### 4.2 节点解析 (parser.py)
- 解析多种格式的节点链接 (ss://, vmess://, trojan://, vless://)
- 支持从 YAML 文件中提取节点

### 4.3 过滤与清洗 (filters.py, cleaner.py)
- **IP 黑名单过滤**: 过滤 `ip-block-list.txt` 中定义的 IP
- **HTTP 协议过滤**: 移除 type=http 的节点
- **SS 特殊清洗**: 移除 cipher=ss 的无效节点 (cleaner.py)

### 4.4 去重与排序 (proxy_tools.py)
- 基于 (server, port) 去重
- 支持手动节点 (名称含 `|M|`) 优先级最高
- 按国家代码和使用次数排序

### 4.5 GeoIP 重命名 (geoip.py)
- 查询 IP 归属地 (国家/城市)
- 生成标准化名称: `🇺🇸 US|美国 01`
- 使用 `Country.mmdb` 和 `City.mmdb` 数据库

### 4.6 节点统计 (csvtool.py)
- 记录每个服务器 IP 的使用次数
- 持久化到 `node-server-statistics.csv`
- 用于生成推荐排序

## 5. 配置文件说明

### 5.1 输入配置

| 文件 | 路径 | 说明 |
|------|------|------|
| sources.json | config/sources.json | 订阅源列表 |
| manual_nodes.yml | config/manual_nodes.yml | 手动节点配置 |
| ip-block-list.txt | config/ip-block-list.txt | IP 黑名单 |
| merge-template.yml | config/templates/clash/merge-template.yml | Clash 配置模板 |

### 5.2 输出文件

| 文件 | 路径 | 格式 |
|------|------|------|
| merge.yml | s/merge.yml | Clash YAML |
| v2ray_sub.txt | s/v2ray_sub.txt | V2RayN Base64 |
| node-server-statistics.csv | s/node-server-statistics.csv | 节点统计 |

### 5.3 资源文件

| 文件 | 用途 |
|------|------|
| Country.mmdb | GeoIP 国家级数据库 |
| City.mmdb | GeoIP 城市级数据库 |
| GeoSite.dat | Clash 内核使用的域名规则 |

## 7. 自动化 (GitHub Actions)

### merge.yml 工作流
- **触发**: 每小时执行 (`5 * * * *`) 或手动触发
- **功能**:
  1. 下载最新 GeoIP 数据库
  2. 执行 cleaner.py 清洗
  3. 执行 merge.py 合并
  4. 检测文件变化
  5. 自动提交推送
  6. 发送 Telegram 通知

### conn.yml 工作流
- 用于连接性测试

## 8. 节点命名规范

```
{国旗} {国家代码}|{国家名}[-{城市名}] {序号:02d}

示例:
🇺🇸 US|美国 01
🇸🇬 SG|新加坡-新加坡 02
🇭🇰 HK|香港-九龙 03
🌐 XX 01 (未知地区)
```

### 特殊节点
- 手动节点: 名称包含 `|M|`
- 时间戳节点: 名称包含 `-Timestamp`

## 9. 使用说明

### 本地运行

```bash
# 激活虚拟环境
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 执行合并 (标准)
python src/merge.py

# 强制执行 (跳过更新检测)
python src/merge.py --force

# 开发者模式 (详细日志)
python src/merge.py --dev
```

### 输出格式
- **Clash**: 导入到 Clash for Windows / ClashX / Meta 系列客户端
- **V2RayN**: 导入到 V2RayN / V2RayNG 客户端

## 10. 注意事项

1. **GeoSite.dat** 由 Clash 内核运行时加载，无需在 Python 侧处理
2. **sub-store** 子项目已与主项目分离，无数据交互
3. **SHA256 校验** 用于判断订阅是否有更新，避免无意义重写
4. **手动节点** 优先级最高，会排除同名 IP 的自动节点
